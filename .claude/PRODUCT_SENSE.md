# PRODUCT_SENSE.md — Interview Connect 제품 비전

`product-analyst`와 `domain-designer`가 설계 결정 시 이 파일을 참조한다.

---

## 핵심 가치 제안

> "면접 경험자와 구직자를 직접 연결하는 플랫폼"

잡플래닛과의 차별점: **정보 제공**이 아닌 **사람 간 연결**.
후기는 연결의 시작점이고, Q&A는 연결의 매개체다.

---

## 수익 모델 (전환 퍼널)

```
비로그인 SEO 유입
    ↓ (후기는 공짜로 다 보여줌 → 검색 상위 노출)
Q&A 답변 블러 발견
    ↓ (이 답변 보고 싶으면 회원가입)
GENERAL 회원
    ↓ (후기 쓰고 싶으면 면접 인증)
VERIFIED 회원 → 후기 작성, 1:1 연결 가능
```

**핵심 원칙:** 비로그인에게 콘텐츠를 최대한 공개해야 검색 유입이 생긴다.
Q&A 답변만 블러 처리하여 가입 동기를 만든다. 이 이외의 콘텐츠를 막으면 안 된다.

---

## 사용자 유형별 가치

### 구직자 (주 타깃)
- 특정 회사 면접 정보를 미리 알고 싶다
- 실제 경험자에게 질문하고 싶다
- 면접 후기 + Q&A를 무료로 열람 가능

### 면접 경험자 (콘텐츠 생산자)
- 자신의 경험을 공유하고 싶다
- 구직자를 도우면서 커뮤니티에 기여
- 인증 회원만 후기 작성 가능 → 신뢰도 보장

---

## 설계 원칙

### 1. SEO First
- 후기/회사 상세 페이지는 비로그인도 전체 내용 읽기 가능
- 서버 사이드 렌더링으로 크롤러가 콘텐츠 인식 가능
- 후기 URL: `/reviews/{id}` — 사람이 읽을 수 있는 메타 정보

### 2. 신뢰 기반 커뮤니티
- 면접 인증(Verification) 후 후기 작성 가능
- 익명 질문 가능하나 답변은 인증 회원만 → 신뢰도 보장
- 1:1 쪽지는 수락제 → 작성자 보호

### 3. 점진적 기능 해제
- 비로그인 → GENERAL → VERIFIED 순서로 기능 확장
- 각 단계에서 명확한 가치를 제공하여 다음 단계 전환 유도

## Phase별 목표

| Phase | 기간 | 핵심 목표 | 성공 지표 |
|-------|------|---------|---------|
| 1 | Week 1~4 | MVP — 후기 공개 + Q&A 블러 전환 | 후기 50건+, 전환율 10%+ |
| 2 | Week 5~8 | 연결 — 1:1 쪽지 + 인증 시스템 | 연결 100건+ |
| 3 | Week 9~12 | 성장 — 검색/SEO/알림/소셜로그인 | MAU 1,000+ |
| 4 | Week 13~16 | 확장 — 멘토링/실시간 채팅 | 멘토 50명+ |

---

## Phase 1 경계 (엄격 준수)

**구현:** 후기 CRUD, Q&A + 블러, 회원가입/로그인, 회사 검색, 알림 기본

**구현 금지 (Phase 2+):**
- 면접 인증 시스템 (Verification)
- 1:1 쪽지 (ChatRoom, ChatMessage)
- S3 파일 업로드
- 소셜 로그인 (Kakao, Google)
- 이메일 알림

---

## 콘텐츠 품질 기준

- 면접 후기는 VERIFIED 회원만 → 허위 후기 최소화
- 질문은 누구나 가능하되 답변은 VERIFIED만 → 답변 신뢰도
- 익명 질문 허용 → 민감한 질문 장벽 낮춤
